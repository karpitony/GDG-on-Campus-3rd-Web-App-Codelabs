# 🧩 Web/App Codelab — Activity 3

## 오프라인에서도 동작하는 PWA 구현하기

이번 Activity 3에서는 웹 애플리케이션을 오프라인 환경에서도 동작하도록 만드는 PWA 기능을 직접 구현합니다.
Service Worker와 Cache Storage API를 활용하여, 네트워크 연결이 없는 상태에서도 페이지가 정상적으로 작동하도록 만드는 경험을 목표로 합니다.

이번 실습은 Progressive Web App(PWA) 코드랩 시리즈 중 첫 번째이며,
앞으로 이어질 8개의 시리즈를 위한 핵심 개념을 다지는 단계입니다.

## 📘 학습 목표

- 직접 서비스 워커 작성
- 기존 웹 애플리케이션에 서비스 워커 추가
- 서비스 워커와 캐시 스토리지 API를 사용하여 리소스를 오프라인에서 사용할 수 있도록 만들기

## 🧱 심화 과제 (선택)

Codelab은 제공된 예제 기반으로 진행되지만, 심화 과제는 실전 적용이 목표입니다.
아래 과제 중 일부만 선택하여 수행해도 됩니다.

---

### 1) 기존 프로젝트에 서비스 워커 적용

- 본인이 진행 중인 개인 프로젝트 또는 팀 프로젝트에 PWA 오프라인 기능 직접 적용
- 프로젝트 특성에 맞는 캐싱 전략 선택 및 변경 이유 기록
  - 예: stale-while-revalidate, cache-first, network-first, cache-only, network-only

### 2) 블로그 또는 기술 문서 작성

- PWA에서 오프라인 기능이 동작하는 원리를 주제로 기술 블로그를 작성합니다.
- 서비스 워커의 요청 흐름을 **직접 구현한 코드 기반으로 설명합니다.**
- 다음을 포함하면 더욱 좋습니다!
  - 네트워크 ON/OFF 동작 비교 캡처
  - 적용한 캐시 전략 설명
  - 마주한 문제와 해결 방법
